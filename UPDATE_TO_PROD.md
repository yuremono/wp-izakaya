# 日常更新の本番反映フロー

このプロジェクトで、管理画面の編集やテーマ修正を本番へ反映するときの標準手順。

## 前提

- 何を変えたかを先に分ける。
- テーマのコードはローカルを正にする。
- 投稿、固定ページ、画像、メニュー、CF7、SEO などの管理画面内容は、どの環境を正にするかを決めてから触る。

## 1. 変更を分類する

- テーマの PHP、SCSS、JS、ACF 登録、CPT 登録を変えたら、コード更新。
- 文言、画像、投稿、固定ページ、メニュー、フォーム文言、SEO 設定を変えたら、管理画面更新。

## 2. ローカルで編集する

1. まず Local で編集する。
2. テーマを触ったら `npm run build` を通す。
3. PHP を触ったら `composer run phpcs` を通す。
4. 管理画面の内容を同期したいときは、必要に応じて `exports/yuremono-wp-content.xml` を出力する。

## 3. 本番へ反映する

### テーマだけ変わった場合

1. `npm run deploy -- --zip-only --no-phpcs` で配布用 ZIP を作る。
2. ZIP を本番へアップロードするか、SSH があるなら `DEPLOY_HOST` / `DEPLOY_USER` / `DEPLOY_PATH` を設定して同期する。
3. 本番でテーマを有効化する。

### 管理画面の内容も変わった場合

1. `exports/yuremono-wp-content.xml` を本番の WordPress インポーターで取り込む。SSH と WP-CLI が使えるなら `npm run deploy:prod` でまとめて反映できる。
2. 表示設定、パーマリンク、メニュー割り当てを確認する。
3. Contact Form 7、SEO SIMPLE PACK、画像パス、リンク先を見直す。

`wp import` は再実行すると投稿や固定ページが増えることがある。XML が本当に更新されたときだけ使う。

### 両方変わった場合

1. まずテーマを反映する。
2. 次にコンテンツを取り込む。
3. 最後に管理画面の個別設定を入れ直す。

この3段階は、`npm run deploy:prod` でまとめられる。テーマ反映は `npm run deploy`、コンテンツ取り込みは本番の WordPress インポーターまたは本番側 WP-CLI が必要。

## 4. 本番確認

- トップページが想定どおり出る。
- 固定ページ、`work`、`news` が見える。
- メニューが正しい位置に入っている。
- フォーム送信が通る。
- HTTPS で開く。
- スマホ表示で崩れていない。
- mixed content が出ていない。

## 絶対にやらないこと

- `DEPLOY_PATH` を確認せずに `npm run deploy -- --delete` を実行する。
- 既存本番 DB に `exports/yuremono-wp-db.sql` を無確認で流す。
- ローカル用の `wp-load.php` パスやデモ投入スクリプトを、対象サイトを確認せずに実行する。
- 送信元メールをローカル用アドレスやユーザー入力メールにする。
- 本番で直した内容を、あとから Local で古い状態で上書きする。

## 迷ったら

- 先に Local で直す。
- どの環境が正かを決める。
- そのあとに反映する。
